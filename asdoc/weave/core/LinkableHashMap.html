requestObject()method 
public function requestObject(name:String, classDef:Class, lockObject:Boolean):*

This function creates an object in the hash map if it doesn't already exist. If there is an existing object associated with the specified name, it will be kept if it is the specified type, or replaced with a new instance of the specified type if it is not.

Parameters
name:String — The identifying name of a new or existing object.
 
classDef:Class — The Class of the desired object type.
 
lockObject:Boolean — If this is true, the object will be locked in place under the specified name.

Returns
* — The object under the requested name of the requested type, or null if an error occurred.

setNameOrder()method 
public function setNameOrder(newOrder:Array):void

This will reorder the names returned by getNames() and the objects returned by getObjects(). Any names appearing in newOrder that do not appear in getNames() will be ignored. Callbacks will be called if the new child order differs from the old order.

Parameters
newOrder:Array — The new desired ordering of names and their corresponding objects.

public function setSessionState(newStateArray:Array, removeMissingDynamicObjects:Boolean):void

This function will update the list of child objects based on an absolute or incremental session state.

Parameters
newStateArray:Array — An Array of child name Strings or DynamicState objects containing the new values and types for child objects.
 
removeMissingDynamicObjects:Boolean — If true, this will remove any child objects that do not appear in the session state.
